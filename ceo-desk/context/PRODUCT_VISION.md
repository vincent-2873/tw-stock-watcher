# 呱呱投資招待所 - 產品願景 v1.0

## 產品定位
台灣第一個「多智能體 AI 投資委員會」,遊戲化呈現。
使用者進入 = 走進日式招待所,看到 7 個有不同投資流派的 AI agents 開會討論,得到經過多方辯論的建議。

## 12 類核心願景

### 1. 商業模式
- B2C 訂閱 + B2B AI 落地展示
- CEO 身份:公司負責人 + 財經老師 + AI 落地工程師
- 時程:1-3 個月品質優先
- 法規:Vincent 公司結構已處理

### 2. 使用者系統
多使用者 + 權限分層(L1 CEO / L2 合夥人 / L3 VIP / L4 付費 / L5 訪客)

### 3. 角色創建系統(遊戲化)
登入後選角色(可愛原創動物)+ 個性(穩健/積極/保守/衝動)+ 能力值(分析力/耐心/抗風險/紀律)+ 專屬角色卡

### 4. AI Agents 辦公室
視覺化「看得見的 AI」、木造日式招待所俯視、Pixel art 侘寂風、點擊 agent 互動

### 5. 戰情室會議系統 ⭐
7 個 agents 根據流派辯論 → 呱呱整合 → 使用者看到完整辯論過程學到投資思維

### 6. 7 個 Agents 流派
見 CHARACTER_DESIGN.md

### 7. 個人化 ETF 建議
輸入金額 → 分析風險屬性/市場/偏好 → 輸出配置 + 理由 + 風險提示

### 8. 呱呱模擬倉 + 回測
呱呱自己下場試錯、公開績效、敢承認失敗(spec 19 精神)

### 9. 視覺設計原則
侘寂風 × 木頭褐色 × Pixel art
色調:原木深褐 #6B4F3F / 柚木棕 #A0826D / 和紙米 #F5EBDC / 秋葉橘 #D4A574 / 苔蘚綠 #7A8B6D / 墨黑 #2C2825
氛圍:溫暖日式老木屋
靈感:星露谷 × 魔女宅急便
IP 紅線:絕對原創

### 10. 技術架構
前台(客戶)+ 後台(Vincent)、WebSocket 即時、整合既有 VSIS 不重建
Next.js + Framer Motion,未來考慮 Phaser.js

### 11. 結構化循環系統(CEO 核心能力)
Vincent 底層方法論:VSIS=投資、另一間=業務、未來所有事業都是結構化循環系統
CEO Desk 本身就是這哲學的實踐

### 12. 協作哲學
Vincent (CEO) + Claude (CTO) + Claude Code (工程主管)
AI 要質疑 CEO 但確認真實狀況、不預設立場、結構化可複製
