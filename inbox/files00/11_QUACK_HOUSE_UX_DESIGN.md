## 1. 品牌定位

### 1.1 品牌名稱

- **中文**：呱呱投資招待所
- **英文**：Quack House
- **副標**：Your Quiet Trading Companion / 一隻安靜陪你看盤的鴨

### 1.2 品牌精神

這不是一個冰冷的股票分析工具，而是一個**有溫度、有態度的投資陪伴者**。

- 🦆 **呱呱** = 不只是吉祥物，是 AI 分析師的化身
- 🏮 **招待所** = 像日式茶室一樣，一個安靜思考的地方
- 🍵 **侘寂** = 拒絕俗豔，欣賞不完美之美

### 1.3 Slogan 候選

主標：
- 🦆「一隻呱呱，陪你想清楚每一筆。」
- 🏮「不給明牌，教你思考。」（延續 VSIS 精神）
- 🍵「靜心看盤，慢慢致富。」

情境副標：
- 盤前：「呱呱泡好茶了，你今天的功課來了。」
- 盤中：「池塘在晃，先別急著跳下去。」
- 盤後：「今天的池塘很熱鬧，呱呱整理好了。」

## 2. 設計哲學

### 2.1 三大原則

#### 原則 1：**留白即態度**
侘寂風的核心是「不塞滿」。
- 每個頁面留出至少 30% 空白
- 不用鮮豔色塊、不用粗框線
- 資訊按「重要性層次」排列，不是堆疊

#### 原則 2：**動態藏在細節**
**靜態畫面 70% + 動態細節 30%**。
- 大面積保持安靜
- 只有在特定元素做微動畫（呱呱搖頭、熱度條流動、數字跳動）
- 動畫要**短、慢、有呼吸感**（不要快閃）

#### 原則 3：**驚喜感 > 功能感**
使用者預期會得到功能，但意外發現小巧思。
- 連續使用 7 天 → 呱呱說「謝謝你每天來茶室」
- 點到漲停股 → 呱呱做慶祝動作
- 連續虧損 → 呱呱遞上一杯茶（不判斷、不說教）

### 2.2 三個避免

#### 避免 1：**太像 Bloomberg Terminal**
- ❌ 滿滿的紅綠數字牆
- ❌ 密密麻麻的表格
- ✅ 用「資訊卡片」分組，有呼吸空間

#### 避免 2：**太像幼稚的 APP**
- ❌ 繽紛彩色
- ❌ 花俏動畫
- ✅ 低飽和、低調、有質感

#### 避免 3：**太像高冷的精品網站**
- ❌ 全黑全白，毫無情緒
- ❌ 照片很美但找不到功能
- ✅ 冷靜之中藏溫度（呱呱就是關鍵）

## 4. 呱呱角色設計

### 4.1 呱呱的身份設定

**背景故事**：
> 呱呱是招待所的住客，也是老闆。他不善言辭，但總在你看盤看累時端一杯茶過來。他懂市場，但更懂節奏。

**個性**：
- 🦆 沉穩、慢條斯理
- 🧘 不恐慌、不激動
- 📚 懂很多但不炫耀
- 🍵 在關鍵時刻遞茶、遞藥、遞建議

### 4.2 呱呱的 8 種情緒（SVG 插畫）

| 情境 | 表情 | 動作 | 語氣 |
|------|------|------|------|
| 😌 **平靜呱呱** | 閉眼淡淡微笑 | 坐著喝茶 | 「今天池塘很安靜。」 |
| 🤔 **思考呱呱** | 歪頭沉思 | 摸下巴 | 「讓我想一下⋯⋯」 |
| 👀 **觀察呱呱** | 瞪大眼 | 舉望遠鏡 | 「有什麼動靜了。」 |
| 😊 **開心呱呱** | 彎彎眼笑 | 舉茶杯 | 「不錯的一筆。」 |
| 😴 **睡覺呱呱** | 閉眼打呼 | 靠著休息 | （盤後顯示）|
| 🤒 **脫水呱呱** | 無力垂頭 | 需要水 | 「今天跌得有點多⋯⋯」 |
| 📚 **研究呱呱** | 戴眼鏡 | 翻報告 | 「這個題材我查查。」 |
| 🧘 **禪定呱呱** | 閉眼盤坐 | 打坐 | 「市場在動，心不動。」 |

### 4.3 呱呱的出現時機

#### 固定出現位置
1. **首頁右上角** → 平常是平靜呱呱
2. **AI 分析師頁面** → 大隻呱呱（主視覺）
3. **載入中畫面** → 思考呱呱 + 呱呱一下 loading 動畫
4. **空狀態** → 睡覺呱呱 + 「還沒有資料，呱呱先睡一下」

#### 動態觸發
- 你打開首頁 → 觀察呱呱從角落探頭
- 點到漲停股 → 開心呱呱舉杯
- 連續虧損 3 天 → 脫水呱呱（輕聲說「喝口茶吧」）
- 晚上 10 點後 → 睡覺呱呱
- 盤前 8:30 → 平靜呱呱遞上「今日功課」
- 連續登入 7 天 → 呱呱送小驚喜（見第 9 節）

### 4.4 呱呱的對話框風格

```
┌────────────────────────────┐
│  🦆                         │
│  呱──                      │  ← 這個「呱」會微微抖動
│                            │
│  「今天池塘有動靜。        │
│   要一起看看嗎？」         │  ← 字慢慢打出來（typing 效果）
│                            │
│            [好啊] [待會]   │
└────────────────────────────┘
```

設計重點：
- 對話框是**和紙紋理**背景
- 邊框用**淡墨水漬**而非直線
- 字是**慢慢打出來**的（typewriter 動畫）
- 尾巴有個小小的**呱呱頭影**指向他

## 5. 資訊架構重組

### 5.1 現有問題診斷

VSIS 現有導航的問題：
```
【現有】: 市場 | 股票 | 回測 | 紙上 | 報告 | 警示 | Chat

問題：
❌ 7 個平行選單，無主次之分
❌ 「紙上」「回測」是低頻功能，卻放在主導航
❌ 首頁不主動告訴你「今天要做什麼」
❌ Chat 藏太深，應該隨時能叫呱呱
```

### 5.2 新導航：**3 主 + 3 輔 + 浮動**

```
【主導航】（每日必用，永遠在側邊/底部）
┌──────────────────────────┐
│  🏠 今日       ← 首頁 Dashboard    │
│  🦆 池塘       ← 題材熱度 + 下鑽   │
│  📓 我的筆記   ← 自選 + 持股 + 觀察 │
└──────────────────────────┘

【輔助導航】（收進「更多」選單）
┌──────────────────────────┐
│  🗺️ 產業地圖               │
│  📊 市場全景               │
│  🛠️ 工具箱（回測/模擬/警示）│
└──────────────────────────┘

【全局浮動】
┌──────────────────────────┐
│  💬 呱呱（右下角固定）     │  ← 任何頁面都能叫出來
└──────────────────────────┘
```

### 5.3 命名哲學：用「侘寂/招待所」語彙取代科技詞

| 原本名稱（科技感） | 新名稱（侘寂感）| 理由 |
|-------------------|----------------|------|
| Dashboard | 🏠 今日 | 每日必訪、簡潔 |
| 題材分析 | 🦆 池塘 | 呱呱在池塘觀察、動靜意象 |
| 自選股 | 📓 我的筆記 | 像日記、隨身冊 |
| 產業分類 | 🗺️ 產業地圖 | 視覺化意象 |
| AI Chat | 💬 呱呱 | 角色化、親切 |
| 回測 | 🛠️ 時光機 | 趣味 |
| 紙上交易 | 🎭 彩排室 | 劇場感 |
| 警示 | 🔔 小鈴鐺 | 日式意象 |
| 報告 | 📜 手札 | 有歷史感 |

## 8. 互動與動態設計

### 8.1 動畫原則

**核心原則：Slow Design**
- 動畫不超過 500ms
- 使用 `ease-in-out` 或自訂貝茲曲線
- 避免「彈跳」「快閃」
- 多用「淡入淡出」「位移」「透明度」

### 8.2 全站必備動畫

#### 動畫 1：**載入狀態**（呱呱一下）

```css
/* 呱呱 loading 動畫 */
@keyframes quack-breathing {
  0%, 100% { transform: scale(1); opacity: 0.8; }
  50%      { transform: scale(1.02); opacity: 1; }
}

.loading-quack {
  animation: quack-breathing 2s ease-in-out infinite;
}
```

對話框：
```
  🦆  呱...  ← 每 1 秒一個呱，疊加
  🦆  呱 呱...
  🦆  呱 呱 呱...
```

#### 動畫 2：**熱度條流動**

```css
@keyframes heat-flow {
  0%   { background-position: 0% 50%; }
  100% { background-position: 100% 50%; }
}

.heat-bar {
  background: linear-gradient(
    90deg,
    var(--accent-gold) 0%,
    var(--heat-high) 50%,
    var(--accent-gold) 100%
  );
  background-size: 200% 100%;
  animation: heat-flow 3s linear infinite;
}
```

#### 動畫 3：**卡片懸浮**

```css
.card {
  transition: all 300ms cubic-bezier(0.2, 0.8, 0.2, 1);
}
```
